fn transfer_to_btc_wallet<'life0, 'async_trait>( &'life0 self, request: Request<Outpoints>, ) -> Pin<Box<dyn Future<Output = Result<Response<RawSignedTx>, Status>> + Send + 'async_trait>>
where Self: 'async_trait, 'life0: 'async_trait,

Sends the given outpoints to the operator’s btc wallet. The transaction will also be broadcasted to the network. Each outpoint must pay to the operator’s taproot address (xonly key, no merkle root). The rpc also checks if any outpoint is the collateral of the operator, and rejects the request if so.

§Parameters
  • outpoints: The outpoints to send to the operator’s btc wallet
§Returns
  • Raw signed tx that transfers the given outpoints to the operator’s btc wallet address

fn internal_withdraw<'life0, 'async_trait>( &'life0 self, request: Request<WithdrawParams>, ) -> Pin<Box<dyn Future<Output = Result<Response<RawSignedTx>, Status>> + Send + 'async_trait>>
where Self: 'async_trait, 'life0: 'async_trait,

Prepares a withdrawal if it’s profitable and the withdrawal is correct and registered in Citrea bridge contract. If withdrawal is accepted, the payout tx will be added to the TxSender and success is returned, otherwise an error is returned. If automation is disabled, the withdrawal will not be accepted and an error will be returned. Note: This is intended for operator’s own use, so it doesn’t include a signature from aggregator.

fn withdraw<'life0, 'async_trait>( &'life0 self, request: Request<WithdrawParamsWithSig>, ) -> Pin<Box<dyn Future<Output = Result<Response<RawSignedTx>, Status>> + Send + 'async_trait>>
where Self: 'async_trait, 'life0: 'async_trait,

First, if verification address in operator’s config is set, the signature in rpc is checked to see if it was signed by the verification address. Then prepares a withdrawal if it’s profitable and the withdrawal is correct and registered in Citrea bridge contract. If withdrawal is accepted, the payout tx will be added to the TxSender and success is returned, otherwise an error is returned. If automation is disabled, the withdrawal will not be accepted and an error will be returned.
