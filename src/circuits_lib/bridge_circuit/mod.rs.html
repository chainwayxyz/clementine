1//! # Bridge Circuit Module
2//!
3//! This module implements the Bridge Circuit for Clementine protocol.
4//! It defines the main entry point, `bridge_circuit`, which executes a comprehensive sequence
5//! of cryptographic verifications to securely validate a payout (fronting) transaction
6//! for a valid peg-out request for an existing peg-in transaction. The circuit
7//! ensures that an operator's claimed Bitcoin chain state is valid and that it has more
8//! cumulative proof-of-work than any challenging watchtower.
9//!
10//! ## Core Workflow
11//! The `bridge_circuit` function orchestrates the entire verification process:
12//! 1.  **Input Reading:** Reads the `BridgeCircuitInput` from the host environment.
13//! 2.  **Header Chain Proof (HCP) Verification:** Validates the operator's proof of the
14//!     Bitcoin header chain.
15//! 3.  **Watchtower Challenge Processing:** Iterates through all submitted watchtower challenges,
16//!     verifying their transaction signatures (`verify_watchtower_challenges`) and their
17//!     accompanying Groth16 proofs of work. It identifies the valid challenger with the
18//!     highest total work (`total_work_and_watchtower_flags`).
19//! 4.  **Work Comparison:** Asserts that the operator's claimed work is greater than the
20//!     maximum work submitted by any valid watchtower.
21//! 5.  **SPV Proof Verification:** Confirms the inclusion of the payout transaction in the
22//!     operator's claimed Bitcoin chain using a Simple Payment Verification (SPV) proof.
23//! 6.  **Light Client & Storage Verification:** Validates the Citrea rollup's state via a light
24//!     client proof (`lc_proof_verifier`) and verifies EVM storage proofs to confirm deposit
25//!     and withdrawal details (`verify_storage_proofs`).
26//! 7.  **Final Output Generation:** Computes a unique `deposit_constant` and a final
27//!     `journal_hash` from critical data points across the proofs. This hash is committed
28//!     to the zkVM journal, serving as the circuit's public, verifiable output.
29//!
30//! ## Key Components and Sub-modules
31//! This module relies on several specialized sub-modules for handling specific cryptographic tasks:
32//! - `groth16` & `groth16_verifier`: For handling Groth16 proof deserialization and verification.
33//! - `spv`: Implements SPV proof logic.
34//! - `lc_proof`: Verifies light client proofs of the rollup state.
35//! - `storage_proof`: Verifies EVM storage proofs.
36//! - `transaction` & `sighash`: Provides utilities for handling Bitcoin transactions and
37//!   computing Taproot sighashes for signature verification.
38//! - `structs`: Defines the data structures used for circuit inputs and outputs.
39//! - `constants`: Contains network-specific constants like method IDs.
40

102/// Executes the bridge circuit in a zkVM environment, verifying multiple cryptographic proofs
103/// related to watchtowers' Bitcoin work, SPV, and storage proofs.
104///
105/// # Parameters
106///
107/// - `guest`: A reference to a zkVM guest implementing `ZkvmGuest`.
108/// - `work_only_image_id`: A 32-byte array representing the work-only image ID used in verification.
109///
110/// # Functionality
111///
112/// 1. Reads the `BridgeCircuitInput` from the host.
113/// 2. Ensures the method ID in `hcp` (header chain proof) matches `HEADER_CHAIN_METHOD_ID`.
114/// 3. Verifies the header chain proof (`hcp`).
115/// 4. Computes maximum total work and watchtower challenge flags using `total_work_and_watchtower_flags`.
116/// 5. Validates that the computed `max_total_work` does not exceed the `total_work` in `hcp.chain_state`.
117/// 6. Fetches the MMR (Merkle Mountain Range) for block hashes from `hcp.chain_state`.
118/// 7. Verifies the SPV proof (`payout_spv`) using the fetched MMR.
119/// 8. Verifies the light client proof using `lc_proof_verifier`.
120/// 9. Ensures the L1 block hash from the light client proof matches the payout transaction's block hash.
121/// 10. Checks storage proofs for deposit and withdrawal transaction indices using `verify_storage_proofs`.
122/// 11. Converts the verified withdrawal outpoint into a Bitcoin transaction ID.
123/// 12. Ensures the withdrawal transaction ID matches the input reference in `payout_spv.transaction`.
124/// 13. Computes the `deposit_constant` using the first OP_RETURN output of the payout transaction.
125/// 14. Extracts and truncates the latest block hash and the payout transaction’s block hash.
126/// 15. Computes a Blake3 hash over concatenated block hash and watchtower flags.
127/// 16. Generates a final journal hash using Blake3 over concatenated data and commits it.
128///
129/// # Panics
130///
131/// - If the method ID in `hcp` does not match `HEADER_CHAIN_METHOD_ID`.
132/// - If `max_total_work` given by watchtowers is greater than `hcp.chain_state.total_work`.
133/// - If the SPV proof is invalid.
134/// - If the storage proof verification fails.
135/// - If the block hash of the light client proof does not match the payout transaction's block hash.
136/// - If the withdrawal transaction ID does not match the referenced input in `payout_spv`.
137pub fn bridge_circuit(guest: &impl ZkvmGuest, work_only_image_id: [u8; 32]) {
138    let input: BridgeCircuitInput = guest.read_from_host();
139    assert_eq!(
140        HEADER_CHAIN_METHOD_ID, input.hcp.method_id,
141        "Invalid method ID for header chain circuit: expected {:?}, got {:?}",
142        HEADER_CHAIN_METHOD_ID, input.hcp.method_id
143    );
144
145    // Verify the HCP
146    guest.verify(input.hcp.method_id, &input.hcp);
147
148    let (max_total_work, challenge_sending_watchtowers) =
149        total_work_and_watchtower_flags(&input, &work_only_image_id);
150
151    let total_work: TotalWork = input.hcp.chain_state.total_work[16..32]
152        .try_into()
153        .expect("Cannot fail");
154
155    // If total work is less than the max total work of watchtowers, panic
156    if total_work < max_total_work {
157        panic!(
158            "Insufficient total work: Total Work {total_work:?} - Max Total Work: {max_total_work:?}",
159        );
160    }
161
162    let mmr = input.hcp.chain_state.block_hashes_mmr.clone();
163
164    if !input.payout_spv.verify(mmr) {
165        panic!(
166            "Invalid SPV proof for txid: {}",
167            input.payout_spv.transaction.compute_txid()
168        );
169    }
170
171    // Light client proof verification
172    let light_client_circuit_output = lc_proof_verifier(input.lcp.clone());
173
174    // Make sure the L1 block hash of the LightClientCircuitOutput matches the payout tx block hash
175    let lc_l1_block_hash = light_client_circuit_output.latest_da_state.block_hash;
176    let spv_l1_block_hash = input.payout_spv.block_header.compute_block_hash();
177
178    if lc_l1_block_hash != spv_l1_block_hash {
179        panic!("L1 block hash mismatch: expected {lc_l1_block_hash:?}, got {spv_l1_block_hash:?}");
180    }
181
182    // Storage proof verification for deposit tx index and withdrawal outpoint
183    let (user_wd_outpoint, vout, move_txid) =
184        verify_storage_proofs(&input.sp, light_client_circuit_output.l2_state_root);
185
186    let user_wd_txid = bitcoin::Txid::from_byte_array(*user_wd_outpoint);
187
188    let payout_input_index: usize = input.payout_input_index as usize;
189
190    assert_eq!(
191        user_wd_txid,
192        input.payout_spv.transaction.input[payout_input_index]
193            .previous_output
194            .txid,
195        "Invalid withdrawal transaction ID"
196    );
197
198    assert_eq!(
199        vout,
200        input.payout_spv.transaction.input[payout_input_index]
201            .previous_output
202            .vout,
203        "Invalid withdrawal transaction output index"
204    );
205
206    let first_op_return_output = get_first_op_return_output(&input.payout_spv.transaction)
207        .expect("Payout transaction must have an OP_RETURN output");
208
209    let round_txid = input.kickoff_tx.input[0]
210        .previous_output
211        .txid
212        .to_byte_array();
213
214    let kickoff_round_vout = input.kickoff_tx.input[0].previous_output.vout;
215
216    let operator_xonlypk: [u8; 32] = parse_op_return_data(&first_op_return_output.script_pubkey)
217        .expect("Invalid operator xonlypk")
218        .try_into()
219        .expect("Invalid xonlypk");
220
221    let deposit_constant = deposit_constant(
222        operator_xonlypk,
223        input.watchtower_challenge_connector_start_idx,
224        &input.all_tweaked_watchtower_pubkeys,
225        *move_txid,
226        round_txid,
227        kickoff_round_vout,
228        input.hcp.genesis_state_hash,
229    );
230
231    let latest_blockhash: LatestBlockhash = input.hcp.chain_state.best_block_hash[12..32]
232        .try_into()
233        .unwrap();
234
235    let payout_tx_blockhash: PayoutTxBlockhash = spv_l1_block_hash[12..32].try_into().unwrap();
236
237    let journal_hash = journal_hash(
238        payout_tx_blockhash,
239        latest_blockhash,
240        challenge_sending_watchtowers,
241        deposit_constant,
242    );
243
244    guest.commit(journal_hash.as_bytes());
245}

247/// Converts a compressed Groth16 proof into a proof structure and verifies it against a given image ID.
248///
249/// # Parameters
250///
251/// - `compressed_proof`: A reference to a 128-byte array containing the compressed Groth16 proof.
252/// - `total_work`: A 16-byte array representing the total accumulated work associated with the proof.
253/// - `image_id`: A reference to a 32-byte array representing the image ID used for verification.
254/// - `genesis_state_hash`: A 32-byte array representing the genesis state hash.
255///
256/// # Returns
257///
258/// - `true` if the Groth16 proof is successfully deserialized and verified.
259/// - `false` if any step in the process fails (e.g., failed deserialization or proof verification).
260///
261/// # Failure Cases
262///
263/// - If deserialization of the compressed proof fails, it returns `false`.
264/// - If Groth16 proof verification fails, it returns `false`.
265fn convert_to_groth16_and_verify(
266    compressed_proof: &[u8; 128],
267    total_work: [u8; 16],
268    image_id: &[u8; 32],
269    genesis_state_hash: [u8; 32],
270) -> bool {
271    let seal = match CircuitGroth16Proof::from_compressed(compressed_proof) {
272        Ok(seal) => seal,
273        Err(_) => return false,
274    };
275
276    let groth16_proof = CircuitGroth16WithTotalWork::new(seal, total_work, genesis_state_hash);
277
278    groth16_proof.verify(image_id)
279}

281/// Verifies watchtower challenge transactions and collects their outputs.
282///
283/// This function performs validation on a set of watchtower challenge transactions
284/// and their associated inputs, witnesses, and public keys. It checks that:
285/// - Each challenge input corresponds to the correct `kickoff_tx` output (P2TR),
286/// - The signature is valid under the Taproot sighash rules,
287/// - The public key matches the one registered for the watchtower,
288/// - And, if all checks pass, it marks the corresponding bit in a 20-byte bitmap
289///   (`challenge_sending_watchtowers`) and collects the first 3 outputs of the
290///   watchtower transaction into `watchtower_challenges_outputs`.
291///
292///   Note: This function only verifies keypath spends.
293///
294/// # Parameters
295/// - `circuit_input`: Data structure holding serialized watchtower transactions, UTXOs, input indices, and pubkeys.
296///
297/// # Returns
298/// A `WatchtowerChallengeSet` containing:
299/// - `challenge_senders`: A 20-byte bitmap indicating which watchtower challenges were valid,
300/// - `challenge_outputs`: A vector of vectors containing the outputs of valid watchtower challenge transactions.
301///   These outputs should conform to the expected structure of either a single OP_RETURN output
302///   or a combination of two P2TR outputs and one OP_RETURN output for the challenge to be
303///   considered when calculating the maximum work). However, it is enough to have a valid signature
304///   to mark the watchtower as a challenge sender.
305///
306/// # Notes
307/// Invalid or malformed challenge data (e.g., decoding errors, invalid signatures)
308/// will be skipped gracefully without causing the function to panic.
309pub fn verify_watchtower_challenges(circuit_input: &BridgeCircuitInput) -> WatchtowerChallengeSet {
310    let mut challenge_sending_watchtowers: [u8; 20] = [0u8; 20];
311    let mut watchtower_challenges_outputs: Vec<Vec<TxOut>> = vec![];
312
313    let kickoff_txid = circuit_input.kickoff_tx.compute_txid();
314
315    if circuit_input.watchtower_inputs.len() > MAX_NUMBER_OF_WATCHTOWERS {
316        panic!(
317            "Invalid number of watchtower challenge transactions: {}",
318            circuit_input.watchtower_inputs.len()
319        );
320    }
321
322    for watchtower_input in circuit_input.watchtower_inputs.iter() {
323        let inner_txouts: Vec<TxOut> = watchtower_input
324            .watchtower_challenge_utxos
325            .iter()
326            .map(|utxo| utxo.0.clone())
327            .collect::<Vec<TxOut>>();
328
329        let prevouts = Prevouts::All(&inner_txouts);
330
331        let watchtower_input_idx = watchtower_input.watchtower_challenge_input_idx as usize;
332
333        if watchtower_input_idx >= watchtower_input.watchtower_challenge_tx.input.len() {
334            panic!(
335                "Invalid watchtower challenge input index, watchtower index: {}",
336                watchtower_input.watchtower_idx
337            );
338        }
339
340        let input = watchtower_input.watchtower_challenge_tx.input[watchtower_input_idx].clone();
341
342        let (sighash_type, sig_bytes): (TapSighashType, [u8; 64]) = {
343            // Enforce the witness to be only 1 element, which is the signature
344            if watchtower_input.watchtower_challenge_witness.0.len() != 1 {
345                panic!(
346                    "Invalid witness length, expected 1 element, watchtower index: {}",
347                    watchtower_input.watchtower_idx
348                );
349            }
350            let signature = watchtower_input.watchtower_challenge_witness.0.to_vec()[0].clone();
351
352            if signature.len() == 64 {
353                (
354                    TapSighashType::Default,
355                    signature[0..64].try_into().expect("Cannot fail"),
356                )
357            } else if signature.len() == 65 {
358                match TapSighashType::from_consensus_u8(signature[64]) {
359                    Ok(sighash_type) => (
360                        sighash_type,
361                        signature[0..64].try_into().expect("Cannot fail"),
362                    ),
363                    Err(_) => {
364                        panic!(
365                            "Invalid sighash type, watchtower index: {}",
366                            watchtower_input.watchtower_idx
367                        );
368                    }
369                }
370            } else {
371                panic!(
372                    "Invalid witness length, expected 64 or 65 bytes, watchtower index: {}",
373                    watchtower_input.watchtower_idx
374                );
375            }
376        };
377
378        let sighash = sighash(
379            &watchtower_input.watchtower_challenge_tx,
380            &prevouts,
381            watchtower_input_idx,
382            sighash_type,
383            watchtower_input.annex_digest,
384        );
385
386        if input.previous_output.txid != kickoff_txid {
387            panic!(
388                "Invalid input: expected input to reference an output from the kickoff transaction (txid: {}), but got txid: {}, vout: {}, watchtower index: {}",
389                kickoff_txid,
390                input.previous_output.txid,
391                input.previous_output.vout,
392                watchtower_input.watchtower_idx
393            );
394        };
395
396        if watchtower_input_idx >= inner_txouts.len() {
397            panic!(
398                "Invalid watchtower challenge input index, watchtower index: {}",
399                watchtower_input.watchtower_idx
400            );
401        }
402
403        let output = inner_txouts[watchtower_input_idx].clone();
404
405        let script_pubkey = output.script_pubkey.clone();
406
407        if !script_pubkey.is_p2tr() {
408            panic!(
409                "Invalid output script type - kickoff, watchtower index: {}",
410                watchtower_input.watchtower_idx
411            );
412        };
413
414        if watchtower_input.watchtower_idx as usize
415            >= circuit_input.all_tweaked_watchtower_pubkeys.len()
416        {
417            panic!(
418                "Invalid watchtower index, watchtower index: {}, number of watchtowers: {}",
419                watchtower_input.watchtower_idx,
420                circuit_input.all_tweaked_watchtower_pubkeys.len()
421            );
422        }
423
424        let pubkey: [u8; 32] = script_pubkey.as_bytes()[2..34]
425            .try_into()
426            .expect("Cannot fail");
427
428        if circuit_input.all_tweaked_watchtower_pubkeys[watchtower_input.watchtower_idx as usize]
429            != pubkey
430        {
431            panic!(
432                "Invalid watchtower public key, watchtower index: {}",
433                watchtower_input.watchtower_idx
434            );
435        }
436
437        let vout = watchtower_input
438            .watchtower_idx
439            .checked_mul(2)
440            .and_then(|x| x.checked_add(circuit_input.watchtower_challenge_connector_start_idx))
441            .expect("Overflow occurred while calculating vout");
442
443        if vout != input.previous_output.vout {
444            panic!(
445                "Invalid output index, watchtower index: {}",
446                watchtower_input.watchtower_idx
447            );
448        }
449
450        let Ok(verifying_key) = VerifyingKey::from_bytes(&pubkey) else {
451            panic!(
452                "Invalid verifying key, watchtower index: {}",
453                watchtower_input.watchtower_idx
454            );
455        };
456
457        let Ok(signature) = Signature::try_from(sig_bytes.as_slice()) else {
458            panic!(
459                "Invalid signature, watchtower index: {}",
460                watchtower_input.watchtower_idx
461            );
462        };
463
464        match verifying_key
465            .verify_prehash(sighash.as_byte_array(), &signature)
466        {
467            Ok(_) => {
468                challenge_sending_watchtowers[(watchtower_input.watchtower_idx as usize) / 8] |=
469                    1 << (watchtower_input.watchtower_idx % 8);
470                watchtower_challenges_outputs
471                    .push(watchtower_input.watchtower_challenge_tx.output.clone());
472            }
473            Err(_) => panic!(
474                "Invalid signature for watchtower challenge with watchtower index: {}, should not happen",
475                watchtower_input.watchtower_idx
476            ),
477        }
478    }
479
480    WatchtowerChallengeSet {
481        challenge_senders: challenge_sending_watchtowers,
482        challenge_outputs: watchtower_challenges_outputs,
483    }
484}

486/// Computes the maximum verified total work and watchtower challenge flags from challenge transactions.
487///
488/// # Parameters
489///
490/// - `circuit_input`: The `BridgeCircuitInput` containing all watchtower inputs and related data.
491/// - `work_only_image_id`: A 32-byte identifier used for Groth16 verification against the work-only circuit.
492///
493/// # Returns
494///
495/// A tuple containing:
496/// - `TotalWork`: The total work from the highest valid watchtower challenge (after successful Groth16 verification).
497/// - `ChallengeSendingWatchtowers`: Bitflags representing which watchtowers sent valid challenges (1 bit per watchtower).
498///
499/// # Notes
500///
501/// - The function robustly skips over any challenges that are malformed, have invalid signatures,
502///   or do not adhere to the expected transaction output structure.
503/// - Each watchtower challenge transaction is expected to contain one of two distinct output structures:
504///     - **Single Output Format:** A single `OP_RETURN` script containing a total of 144 bytes.
505///       This includes the entire 128-byte compressed Groth16 proof followed by the 16-byte `total_work` value.
506///     - **Three Outputs Format:**
507///         - The first two outputs **must** be P2TR (Pay-to-Taproot) outputs. These two outputs
508///           collectively contain the first 64 bytes of the compressed Groth16 proof parts
509///           (32 bytes from each P2TR output).
510///         - The third output **must** be an `OP_RETURN` script, containing the remaining 64 bytes
511///           of the compressed Groth16 proof and the 16-byte `total_work` value.
512/// - Valid commitments are sorted in descending order by their `total_work` value. The Groth16
513///   verifier is then applied sequentially to these sorted commitments, and the first successfully
514///   verified `total_work` is selected as the maximum verified work.
515pub fn total_work_and_watchtower_flags(
516    circuit_input: &BridgeCircuitInput,
517    work_only_image_id: &[u8; 32],
518) -> (TotalWork, ChallengeSendingWatchtowers) {
519    let watchtower_challenge_set = verify_watchtower_challenges(circuit_input);
520
521    let mut valid_watchtower_challenge_commitments: Vec<WatchTowerChallengeTxCommitment> = vec![];
522
523    for outputs in watchtower_challenge_set.challenge_outputs {
524        let compressed_g16_proof: [u8; 128];
525        let total_work: [u8; 16];
526
527        match outputs.as_slice() {
528            // Single OP_RETURN output with 144 bytes
529            [op_return_output, ..] if op_return_output.script_pubkey.is_op_return() => {
530                // If the first output is OP_RETURN, we expect a single output with 144 bytes
531                let Some(Ok(whole_output)) = parse_op_return_data(&op_return_output.script_pubkey)
532                    .map(TryInto::<[u8; 144]>::try_into)
533                else {
534                    continue;
535                };
536                compressed_g16_proof = whole_output[0..128]
537                    .try_into()
538                    .expect("Cannot fail: slicing 128 bytes from 144-byte array");
539                total_work = whole_output[128..144]
540                    .try_into()
541                    .expect("Cannot fail: slicing 16 bytes from 144-byte array");
542            }
543            // Otherwise, we expect three outputs:
544            // 1. [out1, out2, out3] where out1 and out2 are P2TR outputs
545            //    and out3 is an OP_RETURN output with 80 bytes
546            [out1, out2, out3, ..]
547                if out1.script_pubkey.is_p2tr()
548                    && out2.script_pubkey.is_p2tr()
549                    && out3.script_pubkey.is_op_return() =>
550            {
551                let first_output: [u8; 32] = out1.script_pubkey.to_bytes()[2..]
552                    .try_into()
553                    .expect("Cannot fail: slicing 32 bytes from P2TR output");
554                let second_output: [u8; 32] = out2.script_pubkey.to_bytes()[2..]
555                    .try_into()
556                    .expect("Cannot fail: slicing 32 bytes from P2TR output");
557
558                let Some(Ok(third_output)) =
559                    parse_op_return_data(&out3.script_pubkey).map(TryInto::<[u8; 80]>::try_into)
560                else {
561                    continue;
562                };
563
564                compressed_g16_proof =
565                    [&first_output[..], &second_output[..], &third_output[0..64]]
566                        .concat()
567                        .try_into()
568                        .expect("Cannot fail: concatenating and converting to 128-byte array");
569
570                // Borsh deserialization of the final 16 bytes is functionally redundant in this context,
571                // as it does not alter the byte content. It is retained here for consistency and defensive safety.
572                total_work = borsh::from_slice(&third_output[64..])
573                    .expect("Cannot fail: deserializing 16 bytes from 16-byte slice");
574            }
575            _ => continue,
576        }
577
578        let commitment = WatchTowerChallengeTxCommitment {
579            compressed_g16_proof,
580            total_work,
581        };
582
583        valid_watchtower_challenge_commitments.push(commitment);
584    }
585
586    valid_watchtower_challenge_commitments.sort_by(|a, b| b.total_work.cmp(&a.total_work));
587
588    let mut total_work_result = [0u8; 16];
589
590    for commitment in valid_watchtower_challenge_commitments {
591        if convert_to_groth16_and_verify(
592            &commitment.compressed_g16_proof,
593            commitment.total_work,
594            work_only_image_id,
595            circuit_input.hcp.genesis_state_hash,
596        ) {
597            total_work_result = commitment.total_work;
598            break;
599        }
600    }
601
602    (
603        TotalWork(total_work_result),
604        ChallengeSendingWatchtowers(watchtower_challenge_set.challenge_senders),
605    )
606}

608/// Parses the OP_RETURN data from a Bitcoin script. It retrieves the first data push after an OP_RETURN.
609pub fn parse_op_return_data(script: &Script) -> Option<&[u8]> {
610    let mut instructions = script.instructions();
611    if let Some(Ok(Instruction::Op(opcodes::all::OP_RETURN))) = instructions.next() {
612        if let Some(Ok(Instruction::PushBytes(data))) = instructions.next() {
613            return Some(data.as_bytes());
614        }
615    }
616    None
617}

619/// Computes a deposit constant hash using various transaction and cryptographic components.
620///
621/// # Parameters
622///
623/// - `operator_xonlypk`: A 32-byte array representing the operator's X-only public key.
624/// - `watchtower_challenge_connector_start_idx`: A 16-bit unsigned integer marking the start index of the watchtower challenge connector.
625/// - `watchtower_pubkeys`: A slice of 32-byte arrays representing tweaked watchtower public keys.
626/// - `move_txid`: A 32-byte array representing the transaction ID of the move transaction.
627/// - `round_txid`: A 32-byte array representing the transaction ID of the round transaction.
628/// - `kickoff_round_vout`: A 32-bit unsigned integer indicating the vout of the kickoff round transaction.
629/// - `genesis_state_hash`: A 32-byte array representing the genesis state hash.
630///
631/// # Returns
632///
633/// A `DepositConstant` containing a 32-byte SHA-256 hash of the concatenated input components.
634pub fn deposit_constant(
635    operator_xonlypk: [u8; 32],
636    watchtower_challenge_connector_start_idx: u32,
637    watchtower_pubkeys: &[[u8; 32]],
638    move_txid: [u8; 32],
639    round_txid: [u8; 32],
640    kickoff_round_vout: u32,
641    genesis_state_hash: [u8; 32],
642) -> DepositConstant {
643    // pubkeys are 32 bytes long
644    let pubkey_concat = watchtower_pubkeys
645        .iter()
646        .flat_map(|pubkey| pubkey.to_vec())
647        .collect::<Vec<u8>>();
648
649    let watchtower_pubkeys_digest: [u8; 32] = Sha256::digest(&pubkey_concat).into();
650
651    let pre_deposit_constant = [
652        &move_txid,
653        &watchtower_pubkeys_digest,
654        &operator_xonlypk,
655        &watchtower_challenge_connector_start_idx.to_be_bytes()[..],
656        &round_txid,
657        &kickoff_round_vout.to_be_bytes()[..],
658        &genesis_state_hash,
659    ]
660    .concat();
661
662    DepositConstant(Sha256::digest(&pre_deposit_constant).into())
663}

665pub fn journal_hash(
666    payout_tx_blockhash: PayoutTxBlockhash,
667    latest_blockhash: LatestBlockhash,
668    challenge_sending_watchtowers: ChallengeSendingWatchtowers,
669    deposit_constant: DepositConstant,
670) -> blake3::Hash {
671    let concatenated_data = [
672        payout_tx_blockhash.0,
673        latest_blockhash.0,
674        challenge_sending_watchtowers.0,
675    ]
676    .concat();
677
678    let binding = blake3::hash(&concatenated_data);
679    let hash_bytes = binding.as_bytes();
680
681    let concat_journal = [deposit_constant.0, *hash_bytes].concat();
682
683    blake3::hash(&concat_journal)
684}

694/// Computes the Taproot sighash for a given transaction input.
695fn sighash(
696    wt_tx: &Transaction,
697    prevouts: &Prevouts<TxOut>,
698    input_index: usize,
699    sighash_type: TapSighashType,
700    annex_hash: Option<[u8; 32]>,
701) -> bitcoin::sighash::TapSighash {
702    let mut enc = TapSighash::engine();
703    let mut sighash_cache = SighashCache::new(wt_tx);
704    taproot_encode_signing_data_to_with_annex_digest::<_, TxOut, &Transaction>(
705        sighash_cache.borrow_mut(),
706        enc.borrow_mut(),
707        input_index,
708        prevouts,
709        annex_hash,
710        None,
711        sighash_type,
712    );
713    TapSighash::from_engine(enc)
714}

716/// Encodes the BIP341 signing data for any flag type into a given object implementing the
717/// [`io::Write`] trait. This version takes a pre-computed annex hash and panics on error.
718/// Code mostly taken from: https://github.com/rust-bitcoin/rust-bitcoin/blob/9782fa8412e1c767998d018f6c915e51553a83d6/bitcoin/src/crypto/sighash.rs#L619
719pub fn taproot_encode_signing_data_to_with_annex_digest<
720    W: io::Write + ?Sized,
721    T: Borrow<TxOut>,
722    R: Borrow<Transaction>,
723>(
724    sighash_cache: &mut SighashCache<R>,
725    writer: &mut W,
726    input_index: usize,
727    prevouts: &Prevouts<T>,
728    annex_hash: Option<[u8; 32]>,
729    leaf_hash_code_separator: Option<(TapLeafHash, u32)>,
730    sighash_type: TapSighashType,
731) {
732    let tx = sighash_cache.transaction();
733    check_all_prevouts(prevouts, tx);
734
735    let (sighash, anyone_can_pay) = split_anyonecanpay_flag(sighash_type);
736    let expect_msg = "writer should not fail";
737
738    // Epoch
739    0u8.consensus_encode(writer).expect(expect_msg);
740
741    // Control: hash_type (1).
742    (sighash_type as u8)
743        .consensus_encode(writer)
744        .expect(expect_msg);
745
746    // Transaction Data:
747    tx.version.consensus_encode(writer).expect(expect_msg);
748    tx.lock_time.consensus_encode(writer).expect(expect_msg);
749
750    if !anyone_can_pay {
751        // Manually compute sha_prevouts
752        let mut enc_prevouts = sha256::Hash::engine();
753        for txin in tx.input.iter() {
754            txin.previous_output
755                .consensus_encode(&mut enc_prevouts)
756                .expect(expect_msg);
757        }
758        sha256::Hash::from_engine(enc_prevouts)
759            .consensus_encode(writer)
760            .expect(expect_msg);
761
762        // Manually compute sha_amounts
763        let all_prevouts = unwrap_all_prevouts(prevouts);
764        let mut enc_amounts = sha256::Hash::engine();
765        for prevout in all_prevouts.iter() {
766            prevout
767                .borrow()
768                .value
769                .consensus_encode(&mut enc_amounts)
770                .expect(expect_msg);
771        }
772        sha256::Hash::from_engine(enc_amounts)
773            .consensus_encode(writer)
774            .expect(expect_msg);
775
776        // Manually compute sha_scriptpubkeys
777        let mut enc_script_pubkeys = sha256::Hash::engine();
778        for prevout in all_prevouts.iter() {
779            prevout
780                .borrow()
781                .script_pubkey
782                .consensus_encode(&mut enc_script_pubkeys)
783                .expect(expect_msg);
784        }
785        sha256::Hash::from_engine(enc_script_pubkeys)
786            .consensus_encode(writer)
787            .expect(expect_msg);
788
789        // Manually compute sha_sequences
790        let mut enc_sequences = sha256::Hash::engine();
791        for txin in tx.input.iter() {
792            txin.sequence
793                .consensus_encode(&mut enc_sequences)
794                .expect(expect_msg);
795        }
796        sha256::Hash::from_engine(enc_sequences)
797            .consensus_encode(writer)
798            .expect(expect_msg);
799    }
800
801    if sighash != TapSighashType::None && sighash != TapSighashType::Single {
802        // Manually compute sha_outputs
803        let mut enc_outputs = sha256::Hash::engine();
804        for txout in tx.output.iter() {
805            txout.consensus_encode(&mut enc_outputs).expect(expect_msg);
806        }
807        sha256::Hash::from_engine(enc_outputs)
808            .consensus_encode(writer)
809            .expect(expect_msg);
810    }
811
812    // Data about this input:
813    let mut spend_type = 0u8;
814    if annex_hash.is_some() {
815        spend_type |= 1u8;
816    }
817    if leaf_hash_code_separator.is_some() {
818        spend_type |= 2u8;
819    }
820    spend_type.consensus_encode(writer).expect(expect_msg);
821
822    if anyone_can_pay {
823        let txin = tx.tx_in(input_index).expect("invalid input index");
824        let previous_output =
825            get_for_prevouts(prevouts, input_index).expect("invalid prevout for input index");
826        txin.previous_output
827            .consensus_encode(writer)
828            .expect(expect_msg);
829        previous_output
830            .borrow()
831            .value
832            .consensus_encode(writer)
833            .expect(expect_msg);
834        previous_output
835            .borrow()
836            .script_pubkey
837            .consensus_encode(writer)
838            .expect(expect_msg);
839        txin.sequence.consensus_encode(writer).expect(expect_msg);
840    } else {
841        (input_index as u32)
842            .consensus_encode(writer)
843            .expect(expect_msg);
844    }
845
846    if let Some(hash) = annex_hash {
847        hash.consensus_encode(writer).expect(expect_msg);
848    }
849
850    // Data about this output:
851    if sighash == TapSighashType::Single {
852        let mut enc_single_output = sha256::Hash::engine();
853        let output = tx
854            .output
855            .get(input_index)
856            .expect("SIGHASH_SINGLE requires a corresponding output");
857        output
858            .consensus_encode(&mut enc_single_output)
859            .expect(expect_msg);
860        let hash = sha256::Hash::from_engine(enc_single_output);
861        hash.consensus_encode(writer).expect(expect_msg);
862    }
863
864    const KEY_VERSION_0: u8 = 0;
865
866    if let Some((hash, code_separator_pos)) = leaf_hash_code_separator {
867        hash.as_byte_array()
868            .consensus_encode(writer)
869            .expect(expect_msg);
870        KEY_VERSION_0.consensus_encode(writer).expect(expect_msg);
871        code_separator_pos
872            .consensus_encode(writer)
873            .expect(expect_msg);
874    }
875}

877// Helper functions for getting prevouts
878fn get_for_prevouts<'a, T: Borrow<TxOut>>(
879    prevouts: &'a Prevouts<'a, T>,
880    input_index: usize,
881) -> Result<&'a T, PrevoutsIndexError> {
882    match prevouts {
883        Prevouts::One(index, prevout) => {
884            if input_index == *index {
885                Ok(prevout)
886            } else {
887                Err(PrevoutsIndexError::InvalidOneIndex)
888            }
889        }
890        Prevouts::All(prevouts) => prevouts
891            .get(input_index)
892            .ok_or(PrevoutsIndexError::InvalidAllIndex),
893    }
894}
895
896fn unwrap_all_prevouts<'a, T: Borrow<TxOut>>(prevouts: &'a Prevouts<'a, T>) -> &'a [T] {
897    match prevouts {
898        Prevouts::All(prevouts) => prevouts,
899        _ => panic!("cannot get all prevouts from a single prevout"),
900    }
901}
902
903fn check_all_prevouts<T: Borrow<TxOut>>(prevouts: &Prevouts<'_, T>, tx: &Transaction) {
904    if let Prevouts::All(prevouts) = prevouts {
905        if prevouts.len() != tx.input.len() {
906            panic!(
907                "Invalid number of prevouts: expected {}, got {}",
908                tx.input.len(),
909                prevouts.len()
910            );
911        }
912    }
913}

915fn split_anyonecanpay_flag(sighash: TapSighashType) -> (TapSighashType, bool) {
916    match sighash {
917        TapSighashType::Default => (TapSighashType::Default, false),
918        TapSighashType::All => (TapSighashType::All, false),
919        TapSighashType::None => (TapSighashType::None, false),
920        TapSighashType::Single => (TapSighashType::Single, false),
921        TapSighashType::AllPlusAnyoneCanPay => (TapSighashType::All, true),
922        TapSighashType::NonePlusAnyoneCanPay => (TapSighashType::None, true),
923        TapSighashType::SinglePlusAnyoneCanPay => (TapSighashType::Single, true),
924    }

1290    // Helper function to extract and hash the annex
1291    fn get_annex_hash(witness: &Witness) -> Option<[u8; 32]> {
1292        // Using a constant for the annex prefix as defined in bitcoin library
1293
1294        let watchtower_challenge_annex: Option<Annex> = {
1295            if let Some(last_witness_element) = witness.last() {
1296                // Check if the first byte is 0x50 before attempting to create an Annex
1297                if last_witness_element.first() == Some(&TAPROOT_ANNEX_PREFIX) {
1298                    Annex::new(last_witness_element).ok() // Convert Result<Annex, AnnexError> to Option<Annex>
1299                } else {
1300                    None
1301                }
1302            } else {
1303                None
1304            }
1305        };
1306
1307        watchtower_challenge_annex.and_then(|annex| {
1308            let mut enc = sha256::Hash::engine();
1309            match annex.consensus_encode(&mut enc) {
1310                Ok(_) => {
1311                    let hash = sha256::Hash::from_engine(enc);
1312                    Some(hash.to_byte_array()) // Use to_byte_array() for owned array
1313                }
1314                Err(_) => None,
1315            }
1316        })
1317    }
